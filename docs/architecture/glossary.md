# Глоссарий — Knowledge Graph

| Термин | Определение |
|--------|-------------|
| **Note (Заметка)** | Основная сущность системы. Содержит заголовок, содержание, метаданные. |
| **Link (Связь)** | Направленное отношение между двумя заметками. Имеет тип и вес. |
| **Link Type** | Тип связи: `reference`, `dependency`, `related`, `custom`. |
| **Weight (Вес связи)** | Числовое значение (0..1), показывающее силу семантической связи между заметками. |
| **Core Idea (Ядро идеи)** | Набор ключевых слов, отражающих суть заметки. Вычисляется через TF-IDF. |
| **Keyword** | Ключевое слово или фраза, извлечённая из содержания заметки. |
| **Embedding (Эмбеддинг)** | Векторное представление текста размерностью 384 (модель all-MiniLM-L6-v2). |
| **Explicit Link (Явная связь)** | Связь, созданная пользователем (через `[[название]]` или ручное добавление). |
| **Direct Weight** | Вес связи между двумя заметками, вычисленный напрямую (без распространения). |
| **Propagation (Распространение)** | Передача веса от заметки A к заметке C через промежуточные заметки B. |
| **Decay Factor (λ)** | Коэффициент затухания веса при распространении. Значение: 0.5. |
| **Suggestion (Рекомендация)** | Заметка, предложенная пользователю как семантически связанная с текущей. |
| **CQRS** | Command Query Responsibility Segregation — разделение операций изменения и чтения. |
| **Command** | Операция, изменяющая состояние системы (CreateNote, AddLink). |
| **Query** | Операция, только читающая данные (GetSuggestions, GetGraphData). |
| **Domain Event** | Событие, которое произошло в домене (NoteCreated, LinkAdded). |
| **Event Bus** | Шина для публикации и подписки на доменные события. |
| **Specification** | Паттерн для инкапсуляции логики фильтрации (пресет для поиска). |
| **Repository** | Паттерн для доступа к данным. Интерфейс в Domain, реализация в Infrastructure. |
| **Value Object** | Неизменяемый объект, идентифицируемый по значениям полей (Title, Content). |
| **Aggregate (Агрегат)** | Группа объектов, связанных через корневую сущность (Note, Link). |
| **Read Model** | Модель данных, оптимизированная для запросов (может отличаться от модели записи). |
| **Eventual Consistency** | Гарантия, что данные станут консистентными спустя некоторое время. |
| **Outbox** | Таблица для гарантированной публикации событий (в будущем). |
| **Asynq** | Библиотека для асинхронной обработки задач на Go (Redis-based). |
| **pgvector** | Расширение PostgreSQL для работы с векторными эмбеддингами. |
| **TF-IDF** | Статистическая мера важности слова в документе относительно коллекции. |
| **BFS** | Поиск в ширину — алгоритм обхода графа. |
| **Saga** | Паттерн управления распределёнными транзакциями (в будущем). |
| **ATAM** | Architecture Tradeoff Analysis Method — метод анализа архитектурных компромиссов. |
